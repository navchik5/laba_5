## 1 задача
Подсчитать количество пар противоположных чисел среди компонент исходного файла. 
### Алгоритм решения:
1. Создание исходного файла:
  - Открыть бинарный файл для записи.
  - Сгенерировать заданное количество случайных целых чисел в диапазоне.
  - Записать каждое число в файл в бинарном формате.
2. Подсчет пар противоположных чисел:
  - Открыть созданный файл для чтения в бинарном режиме.
  - Прочитать все числа из файла в одномерный массив.
  - Использовать двойной цикл для проверки всех возможных пар чисел:
  - Внешний цикл проходит по всем элементам массива от первого до предпоследнего.
  - Внутренний цикл проходит по всем элементам, начиная со следующего после текущего элемента внешнего цикла.
  - Если сумма двух чисел равна нулю (одно число является отрицательным другого), увеличить счетчик пар.
- Вернуть значение счетчика.

  ## 2 задача
Скопировать элементы заданного файла в квадратную матрицу размером n×n (если элементов файла недостает, заполнить оставшиеся элементы матрицы нулями). Заменить все элементы каждого столбца на максимальный элемент в этом столбце. 
  ### Алгоритм решения:
1. Подготовка исходных данных:
  - Создать бинарный файл, содержащий n×n случайных целых чисел.
2. Чтение данных и формирование матрицы:
  - Прочитать числа из файла в одномерный массив data.
  - Создать квадратную матрицу matrix размером n×n.
  - Заполнить матрицу данными из массива data построчно:
    - Если в data достаточно элементов, скопировать их в матрицу.
    - Если элементов не хватает, оставшиеся ячейки матрицы остаются нулевыми.
3. Обработка матрицы:
  - Для каждого столбца j от 0 до n-1:
    - Найти максимальный элемент в столбце j.
    - Заменить все элементы столбца j на найденное максимальное значение.
  - Вывести исходную и преобразованную матрицы.

## 3 задача
Информация о багаже пассажира описывается массивом, где каждый элемент содержит название единицы багажа (чемодан, сумка, коробка и т.д.) и ее массу. Дан файл, содержащий сведения о багаже нескольких пассажиров. На сколько багаж максимальной массы отличается от багажа минимальной массы? 
### Алгоритм решения:
1. Структура данных:
  - Определить структуру BaggageItem с полями: название (string) и масса (double).
  - Данные о пассажире: массив структур BaggageItem.
2. Создание тестового файла:
  - Для каждого пассажира случайным образом определить количество единиц багажа.
  - Для каждой единицы багажа случайно выбрать название из списка и массу.
  - Записать в бинарный файл:
    - Количество единиц багажа пассажира (int).
    - Для каждой единицы: длину названия (size_t), само название (char*), массу (double).
3. Обработка данных:
  - Прочитать файл последовательно:
    - Прочитать количество единиц багажа очередного пассажира.
    - Для каждой единицы прочитать название и массу, вычислить общую массу багажа пассажира.
    - В процессе чтения поддерживать две переменные: maxMass и minMass.
    - Для каждого пассажира обновлять maxMass и minMass при необходимости.
4. Результат:
  - Вернуть разницу: maxMass - minMass.

## 5 задача
В текстовом файле хранятся целые числа по несколько в строке, исходный файл необходимо заполнить случайными данными. Вычислить максимальный элемент. 
### Алгоритм решения:
1. Подготовка данных:
  - Создать текстовый файл, содержащий несколько строк.
  - В каждой строке разместить от 2 до 6 случайных целых чисел в диапазоне [0, 99], разделенных пробелами.
2. Поиск максимума:
  - Открыть файл для чтения.
  - Инициализировать переменную max значением INT_MIN (минимальное возможное целое).
  - Читать числа из файла последовательно, независимо от разбиения на строки.
  - Для каждого прочитанного числа сравнивать его с текущим max.
  - Если число больше max, обновить значение max.
  - После обработки всех чисел вернуть значение max.

## 6 задача
в текстовом файле хранится текст. Переписать в другой файл строки, оканчивающиеся на заданный символ. 
### Алгоритм решения:
1. Создание исходного текстового файла:
  - Сгенерировать несколько строк текста, каждая из которых состоит из случайного количества слов.
  - Слова выбираются из фиксированного набора английских слов.
2. Фильтрация строк:
  - Запросить у пользователя символ для фильтрации.
  - Открыть исходный файл для чтения и целевой файл для записи.
  - Читать исходный файл построчно.
  - Для каждой строки:
    - Проверить, не пуста ли строка.
    - Проверить, совпадает ли последний символ строки с заданным символом.
    - Если оба условия выполнены, записать строку в целевой файл.
  - Вывести информацию о количестве скопированных строк.
